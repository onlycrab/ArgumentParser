This is a example of using custom rules.
You can create own rules for arguments and check is it satisfied.

First, create a class that extends Rule and override the method isSatisfied().
Then add to the rule the arguments that should execute it by the method addTarget(Argument).
Then add rules to storage object by ArgumentsStorage#addRule(Rule).
After parsing, call the ArgumentsStorage#isRulesSatisfied(): the result will be false if any rule returns false from Rule#isSatisfied(), otherwise the result will be true.

Demonstrator can check if value is an email.
Available arguments : e(email).
Try this:
  -e some@example.com
  -e notAnEmail
  -e @error.org
